# Skill Watcher Log Schema

Skill Watcher writes one redacted JSON object per line to `$CODEX_HOME/skill-watcher/logs/events.jsonl`.

Core fields:

- `schema_version`: current value is `1`.
- `event_id`: UUID generated by the collector when absent.
- `timestamp`: UTC ISO-8601 timestamp generated by the collector when absent.
- `agent`: defaults to `codex`.
- `event_type`: lifecycle label such as `skill_start`, `tool_use`, `skill_stop`, `user_feedback`, or `failure`.
- `workspace`: workspace path or identifier.
- `session_id`: Codex session identifier when available.
- `skill_name`: target skill name.
- `skill_version`: target skill version when known.
- `trigger_reason`: short reason the skill was used.
- `tools_used`: array of tool or command names.
- `files_touched`: array of paths, without file contents.
- `outcome`: `success`, `failure`, `partial`, or `unknown`.
- `failure_type`: optional category such as `tool_error`, `wrong_assumption`, `missed_validation`, `format_error`, or `user_correction`.
- `user_feedback`: short factual feedback.
- `tests_or_checks`: array of validation actions.
- `notes`: short factual trace.

Collectors should avoid full prompts, file contents, complete command transcripts, secrets, and private business data.
